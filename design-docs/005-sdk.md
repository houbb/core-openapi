# 一、定位

前面几个阶段：

```text
Phase 0 API Foundation
        |
        |  管理 API 资产

Phase 1 API Gateway
        |
        |  统一入口

Phase 2 API Definition
        |
        |  描述 API 能力

Phase 3 API Key
        |
        |  身份认证
```

现在：

> API 已经可以调用，但是开发者体验还不够好。

直接调用：

```http
POST /api/v1/ai/chat

Authorization: Bearer sk_xxxxx

Content-Type: application/json

{
 "message":"hello"
}

```

每个开发者都需要：

* 拼 URL
* 管理 Header
* 处理异常
* 解析返回
* 重试
* 日志

SDK Runtime 解决：

> 把 API 变成开发者可以直接使用的代码能力。

---

# 二、核心目标

最终体验：

没有 SDK：

```java
HttpClient client=new HttpClient();

request.setHeader(
"Authorization",
"Bearer sk_xxxx"
);

request.post("/api/v1/chat");

```

有 SDK：

```java
CoreAI ai =
    CoreClient.builder()
        .apiKey("sk_xxxx")
        .build();


ChatResponse response =
    ai.chat(
       "hello"
    );

```

# 5. Error Runtime ⭐⭐⭐⭐⭐

统一异常。

不要：

```java
Exception

```

应该：

```java
CoreException

```

例如：

```java
try{

client.ai.chat();

}

catch(
RateLimitException e
){

}

```

---

错误分类：

```text
AuthenticationError

PermissionError

ValidationError

NetworkError

ServerError

RateLimitError

```

# 十二、关键设计原则

## 1. SDK不是简单HTTP封装

低级：

```text
HTTP Client Wrapper

```

高级：

```text
Developer Experience Layer

```

---

## 2. API Definition必须足够丰富

SDK质量取决于：

```text
API Definition质量

```

所以：

Phase 2 是核心。

---

## 3. 不要绑定语言

不要：

```text
Java SDK优先

```

应该：

```text
API Contract First

```

任何语言都生成。
