# 一、定位

前面的阶段完成了：

```text
Phase 0 API Foundation
        |
        | API资产管理

Phase 1 API Gateway
        |
        | API统一入口

Phase 2 API Definition
        |
        | API文档契约

Phase 3 API Key
        |
        | 调用身份

Phase 4 SDK
        |
        | 开发体验

Phase 5 Security
        |
        | 安全治理

Phase 6 Rate Limit
        |
        | 流量治理

```

但是这些能力目前还是：

> 平台内部管理员能力。

Developer Portal 的目标：

> 把内部 OpenAPI 能力转换成开发者可以自助使用的生态入口。

类似：

Stripe Developer Dashboard。

# 三、设计原则

## 1. 自助化

以前：

开发者：

```text
联系管理员

↓

申请Key

↓

人工配置

```

以后：

```text
注册

↓

创建

↓

调用

```

---

## 2. 产品化

API不再是技术接口。

而是：

```text
API Product

```

例如：

不是：

```
POST /ai/chat
```

而是：

```
AI Conversation API
```

---

## 3. 开发者体验优先

优秀平台：

不是功能最多。

而是：

> 第一次调用成功最快。

# 完成 Phase 7 后：

`core-openapi` 从一个：

> API基础设施

升级成为：

> 开发者生态平台。

此时你的 Core Platform 开始具备类似：

* 云厂商 API 平台
* Stripe 开发者平台
* OpenAI Developer Platform

的雏形。

下一阶段 **Phase 8：API Marketplace Runtime** 才会进一步把 API 从「能力」升级成「商品和生态资产」。
